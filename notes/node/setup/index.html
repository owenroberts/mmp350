We don't have admin priveleges on the classroom machines, so we need to set up a special folder for our software in order to have a working development environment. This should be relatively straight forward but we'll take some time to make sure everyone has this working.

If you're working on home computer or laptop, you can simply download and install NodeJS like any other program. Click here for download. Then skip to the last slide to make sure it worked.

Start by opening terminal.

From any directory, execute the following commands:

git clone --depth=1 https://github.com/Homebrew/homebrew.git ~/.brew

This installs a package manager called Homebrew into our user directory. This will make it easy for us to install NodeJS and other software to our user directory instead of the root directory.

Use this command to open ~/.bash_profile:

open ~/.bash_profile

It should open in TextEdit. If the terminal reponds with The file ... does not exist. use this command to create it:

touch ~/.bash_profile

Then use open again.

Copy and paste the following line into the text document, save it and quit TextEdit.

export PATH="$HOME/.brew/bin:$HOME/.brew/sbin:$PATH"

This line tell the terminal where to find Homebrew because it is an unconventional folder.

Finally, use this command to load the new file:

source ~/.bash_profile

Okay, great. Now lets upate Homebrew and install Node.

brew update
brew doctor
brew install node

That last command is going to take a while. Open a new terminal window and try some Unix practice.

Once Node is installed, run these commands. You should see something similar to the comments returned from the terminal.

which node 
/Users/username/.brew/bin
node --version
node -v
v6.11.1
npm -v
3.10.10

This will tell you if node was successfully installed. We also need npm which is the node package manager, a tool to downloading libraries and dependencies we will need to build a website.

This process might not be perfectly smooth at first but we will make sure every one has it working before moving on.
