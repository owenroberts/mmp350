This semester we will be using Git and GitHub as tools to collaborate on the web apps we will be building.

Today we'll go over some of the basics of using Git before using it to make a copy of the project repo.

Git Repo

Since we'll be working on the same project, I'm going to start by adding my code to a project repo.

Repo is shorthand for repository. It basically means a folder with a bunch of code in it that can be copied and send to different locations.

I'll start by adding a new Git repo in my project folder using the init command.

$ git init

This adds a hidden .git directory in my project folder that keeps track of all of my files and code.

I can use $ git status to check that the repo exists. Use this command a lot while doing things in Git.

Now that I have a repo, I want to create a .gitignore to ignore any files I don't want in the website.

For now that's just the .DS_Store file the Mac OS creates in each folder.

Add and Commit

Now I'm ready to make my first commit.

Making a commit is a process that you will do over and over.

A commit represents a finished verion of a piece of code that you want to save. It could be just a couple lines of CSS or a hundred lines of JavaScript that implement a new feature.

The commit process involves two parts: add and commit.

$ git add .

This command will add all the new or modified files to the git repo. The files will turn green when you do git status.

$ git commit -m "initial commit"

This creates a new commit. You can view the commit using $ git log. To publish your code there must be at least one commit.

Follow this same process for any future updates. Add the files and then commit them.

Push and Pull

Add and Commit is the local part of the process, it's how you prepare code on your computer to be shared.

To share your code you push it to a remote repository and then pull changes that other developers add.

To push and pull, first we need a remote, a location that git saves to send the code to and get it from.

On GitHub, I'm going to create a new repo and name it after the project. Then I'll copy the URL for this repo to use as my remote.

$ git remote add origin GITHUB_URL

To check my remotes I can use the command:

$ git remote -v
