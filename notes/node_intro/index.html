NodeJS

JavaScript was originally written and popularized as a front-end or client-side programming language. NodeJS is an implementation of JavaScript based on the Chrome V8 JavaScript Engine. It allows programmers to create server-side programs written in JavaScript.

NodeJS has become popular because web programmers are familiar with JavaScript and because it uses events and asynchronous I/O to make highly scalable web applications.

Client vs. Server

One of the main topics of this class is the separation of client side and server side programs used by web based applications.

Every time a website or information is accessed through the Internet it involves an interaction between a client, or user, and server, or centralized program.

We're going to learn how to send and receive requests from a server and render the information on a client or browser.

Servers

A server is basically code running on computer which serves information to different users. A server can be any computer but usually it is a powerful computer or network of computers.

Web servers are designed to receive request using specific protocols. A protocol is sort of like a language or series or steps that both computers have agreed on the meaning of. HTTP is one protocol that is often used. Servers also use specific file types like HTML, XML, JSON and media types like .gif, .png, and .jpg.

There a many different programming languages used to create servers, like PHP, Ruby, Python, Java and now JavaScript. The program will open a socket on the server to accept requests from clients. The socket is like a doorway with an address, allowing clients to request contents from the server machine.

Clients

A typical client program is a web browser. A web browser takes a request from a user, retrieves content from a server, and renders that content for the user to view and interact with.
