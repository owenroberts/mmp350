We've all looked at the Adobe Kuler site for making color schemes. But what do the different Color Rules (analogous, monochromatic, triad, complementary) signify? These are basic concepts in color theory. Let's take a look.

The main thing I want to take away from these concepts is the fact that color relationships can be described with numbers.

Because all colors are mapped on the color wheel, we can set up basic relationships and apply them to different colors.

For a complementary color, we just have to move the color wheel 180 degrees.

Hue

Triadic color scheme

Let's look at a working example and then break down the code. This example uses a triadic color scheme. By adjusting the primary color hue, saturation or lightness, we will see all of the colors adjust along with it.

I skate to where the puck is going to be, not to where it has been.
by Wayne Gretzky on Jan 1, 1999 
.posts {
	--hue: 0;
	--sat: 50;
	--lit: 50;
}

How does this work?

We can use CSS custom properties to take a single value for the hue and use it to define three separate colors.

Custom properties can be defined in any CSS selector block using two dashes before the property name. Any CSS value is then assigned to the property.

var

We can start with --hue.

.post { 
	--hue: red;
}

And use it to define a color, using the var() rule.

.post {
	--hue: red;
	--main: var(--hue);
}

When --hue is changed, the --main-color will also change.

Instead of a default color, we will assign a number from the color wheel to the --hue value. Let's start with a nice green.

.post {
	--hue: 120;
}

To create an HSL color, we also need the saturation and lightness values.

.post {
	--sat: 50;
	--lit: 50;
}

calc

Then we need to convert those values to use the % to be read correctly by the CSS hsl() rule. To do this we can use the CSS calc function to do basic arithmetic.

.post {
	--saturation: calc(var(--sat) * 1%);
	--lightness: calc(var(--lit) * 1%);
}

Now we can construct and HSL value.

.post {
	--main-color: hsl(var(--main), var(--saturation), var(--lightness));
}

To define the colors using relationships between the hue, we can use calc again to define a new relationship.

Let's try a complimentary color.

.post {
	--comp: calc(var(--main) + 180);
	--comp-color: hsl(var(--comp), var(--saturation), var(--lightness));
}

Now we can update the entire color scheme by updating a single value.

.post {
	--hue: 240;
}

Here's the resulting complementary color scheme.

I skate to where the puck is going to be, not to where it has been.
by Wayne Gretzky on Jan 1, 1999 
.posts {
	--hue: 240;
	--sat: 50;
	--lit: 50;
}

The rest of the CSS rules.

.posts {
	--hue: 240;
	--sat: 50;
	--lit: 50;
	--main: hsl(var(--comp-hue), calc(var(--sat) * 1%), calc(var(--lit) * 1%));
	--complementary: hsl(calc(var(--hue) + 180), calc(var(--sat) * 1%), calc(var(--lit) * 1%));;
}
.posts .post {
	border: 2px solid var(--main);
}
.posts .post-text {
	color: var(--main);
}
.posts .post-author, .posts .post-date {
	color: var(--complementary);
}

That covers creating a complementary scheme, with complementary color difference of 180 degrees.

The monochromatic CSS rules.

.posts {
	--hue: 240;
	--sat: 50;
	--lit: 50;
	--main: hsl(var(--hue), calc(var(--sat) * 1%), calc(var(--lit) * 1%));
	
	/* update saturation in one */
	--saturation: calc(var(--sat) + 20);
	--one: hsl(calc(var(--hue)), calc(var(--saturation) * 1%), calc(var(--lit) * 1%));;

	/* change lightness in two */
	--lightness: calc(var(--lit) - 20);
	--two: hsl(calc(var(--hue)), calc(var(--sat) * 1%), calc(var(--lightness) * 1%));;
}
.posts .post {
	border: 2px solid var(--main);
}
.posts .post-text {
	color: var(--one);
}
.posts .post-author, .posts .post-date {
	color: var(--two);
}
