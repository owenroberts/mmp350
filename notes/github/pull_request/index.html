Fork

Pull requests can be made between branches on the same project or from a Fork of a project.

We're using Forks from the original project to develop new features.

If you've forked a repository and made changes to the fork, you can ask that the upstream repository accept your changes by creating a pull request.

This way the original project can incorporate new changes.

Pull request

First, navigate to the repo in your GitHub account.

To make a pull request, you must first make changes and commit them to the project. If you have done so, you will see a message showing that your branch is ahead of the original repo.

There, you will find a button to begin a pull request.

This will bring up a window showing the differences between your Fork and the original.

Once you confirm the changes, click the "Create pull request" button.

Then you will be prompted to add a message. This is important to explain to the maintainer what your new changes will do to change the project.

Click "Create pull request" again and it will open the new pull request.

GitHub will create a message area for other contributors to respond to what is changed in the request.

Maintainer

The project maintainer will recieve the pull request.

The maintainer can then review the code and choose to merge the request, clsoe it or add comments for the original requester.
