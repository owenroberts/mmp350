Fork a Repo ~ GitHub Tutorial

A fork is a copy of a repo. Creating a fork allows a user to make their own version of a piece of software while keeping the core functionality of the software up to date with the original project.

Start by logging into GitHub and then navigating to my BMCC GitHub page github.com/owenbmcc.

Find the repo we created in class and then click the Fork button in the upper right hand corner.

This will make a copy of the repo in your own GitHub account.

To download the copy to your computer, open Terminal and clone the repo.

$ cd Desktop/mmp350
$ git clone https://github.com/USER_NAME/REPO_NAME_HERE
$ cd REPO_NAME_HERE

Once the fork is cloned, print out the remotes to make sure it points to your GitHub account.

$ git remote -v

After forking, the origin repo will be on your GitHub account.

To keep things up to date with the original, set the upstream repo to the repo I created in class.

$ git remote add upstream https://github.com/owenbmcc/REPO_NAME_HERE

Then you can grab updates in the code with this command.

$ git pull upstream master
